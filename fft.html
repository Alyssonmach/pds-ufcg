Vamos definir o algoritmo Fast Fourier Transform (FFT), um algoritmo que nos ajuda a entender o comportamento de vrios sinais no domnio da frequncia. A ideia da FFT conseguir calcular as frequncias de um sinal de uma forma rpida e eficiente, utilizando uma estrutura matemtica que reduz drasticamente o nmero de operaes necessrias para alcanar seu objetivo, o tornando um algoritmo extremamente eficiente e utilizando em diversas reas de aplicaes.

Vamos definir algumas consideraes sobre esse algoritmo, baseado em todas as explicaes feitas em sala de aula. Primeiramente, a definio matemtica da FFT dada por:

$X[k] = \sum_{r=0}^{\frac{N}{2} - 1} X_{e}(n)(W_{\frac{N}{2}})^{kr} + W_{N}^{k}\sum_{r=0}^{\frac{N}{2} - 1} X_{o}(n)(W_{\frac{N}{2}})^{kr}$

Sendo $X[k]$ o resultado da FFT, $x(n)$ o sinal no domnio do tempo e $x_e(n)$ os nmeros pares e $x_o(n)$ os ndices mpares do sinal $x(t)$.

Desse modo, podemos definir:

$X[k] = X_e[k] + W_{N}^{k}X_o[k]$

Essa ltima equao ser til na definio do nosso algoritmo. Agora, vamos considerar a estrutura de grafo abaixo, chamada de diagrama de borboleta pelo seu formato. Em cada estgio, as operaes so organizadas de forma que os clculos sejam realizados em pares de valores do sinal. Isso chamado de "diagrama de borboleta" porque os valores so combinados e recalculados de uma maneira que lembra as asas de uma borboleta se movendo.

No exemplo abaixo, o sinal tem tamanho 8, e pela definio, sero necessrios $log_2(8) = 3$ estgios para realizar toda a operao.

Vamos esboar agora como poderiamos definir a o algoritmo da FFT, de forma eficiente:

  1. Obtenha a dimenso do sinal de entrada $x(t)$, de modo a obter um tamanho $N$.

  2. Verifique se $N$ potncia de 2. Pela estrutura do diagrama de borboleta, essa condio necessria para se possa reproduzir a mesma estrutura com diferentes valores de $N$.

  3. Caso a condio 2 seja verdadeira, divida $x(t)$ pelos elementos de indices pares $x_e(t)$ e ndices mpares $x_o(t)$.

  4. Agora vamos implementar uma recurso, as duas partes $x_e(t)$ e $x_o(t)$ so passadas recursivamente pela prpria funo. Isso anlogo recurso na FFT, onde cada sub-sinal processado novamente pela FFT.

  5. Aps as chamadas recursivas, os resultados das transformadas so combinados. Aqui, vemos a etapa onde as "borboletas" do diagrama so formadas. As partes transformadas so combinadas de acordo com as operaes especficas da FFT.

  6. Finalmente, o resultado combinado retornado. Esse resultado a transformada de Fourier do sinal de entrada original.

def verify_power_2(x):
    '''
    Funo que verifica se um nmero  potncia de 2.

    Inputs:
        x (int) -> nmero a ser verificado.
    
    Returns:
        flag_verify (bool) -> retorna True caso o nmero seja potncia de 2, caso contrrio, False.
    '''

    # Se o nmero for negativo, retorna False
    if x <= 0:
        return False
    # Verifica se o nmero  divisvel por 2
    while x % 2 == 0:
        x = x // 2

    # Verifica se o resultado final  1, o que indica que era uma potncia de 2
    flag_verify = x == 1
    return flag_verify

def my_fft(x, view_recursion = False):
    '''
    Funo que calcula a FFT a partir de um sinal x(t).

    Inputs:
        x (list) -> lista contendo os valores numricos do sinal.
        view_recursion (bool) -> flag para mostrar na tela as etapas da recurso acontecendo.
    
    Returns:
        fft (list) -> clculo da FFT a partir do sinal x(t).
    '''

    # Obtm a dimenso do vetor de entrada
    N = len(x)

    # Verifica se o vetor de entrada  potncia de 2
    if not verify_power_2(N):  raise RuntimeError('O nmero no  potncia de 2')

    # Caso no processo interativo o tamanho do vetor for 1 (parada da recurso), retorna esse valor
    if N == 1: return x

    # Separa os elementos pares e mpares da lista, para implementar o grafo de estrutura de borboleta da FFT
    xe, xo = list(), list()
    for i, value in enumerate(x):
        if i % 2 == False: xe.append(value)
        else: xo.append(value)
    
    # Implementa a recurso, semelhante ao processo ilustrado no grafo da estrutura de borboleta da FFT
    xe = my_fft(xe, view_recursion)
    xo = my_fft(xo, view_recursion)

    # Visualizar as etapas da recurso acontecendo
    # Dica: coloque o vetor [0,1,2,3,4,5,6,7] na entrada, fica mais fcil acompanhar os ndices
    if view_recursion:
        print(f'---\nx = {np.round(x, 2)}\n xe = {np.round(xe, 2)}\n xo = {np.round(xo, 2)}')

    # Calcula o somatrio da FFT separando os elementos em N/2 por simetria
    n = np.arange(N)
    Wn = np.exp((-1j) * ((2 * np.pi) / N) * n)

    first_sequence = xe + Wn[:int(N/2)] * xo
    second_sequence = xe + Wn[int(N/2):] * xo

    actual_output = np.concatenate([first_sequence, second_sequence])

    return actual_output

Vamos discutir esses resultados obtidos:

  1. Inicialmente, o sinal dividido em pares e mpares: $[0, 1, 2, 3, 4, 5, 6, 7]$ dividido em $[0, 2, 4, 6]$ $x_e(t)$ e $[1, 3, 5, 7]$ $x_o(t)$.

  2. Em seguida, cada subconjunto dividido novamente em pares e mpares: $[0, 2, 4, 6]$ dividido em $[0, 4]$ $x_e(t)$ e $[2, 6]$ $x_o(t)$, e $[1, 3, 5, 7]$ dividido em $[1, 5]$ $x_e(t)$ e $[3, 7]$ $x_o(t)$.

  3. Cada subconjunto processado recursivamente pela funo my_fft, representando a recurso da FFT. Isso mostrado nos resultados intermedirios onde $x_e(t)$ e $x_o(t)$ so processados separadamente.

  4. Depois que as recurses so concludas, os resultados so combinados. Aqui onde ocorre a formao das "borboletas". Por exemplo, $[0, 4]$ e $[2, 6]$ so combinados para formar $[4+0j, -4+0j] e [8+0j, -4+0j]$.

  5. Finalmente, os resultados combinados so retornados como a transformada de Fourier do sinal de entrada original, $[12+0j, -4+4j, -4+0j, -4-4j, 16+0j, -4+4j, -4+0j, -4-4j]$.
